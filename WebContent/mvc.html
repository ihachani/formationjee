MVC Overview

Model:

Represents the underlying data Database table, list, document, etc. Model is retrieved for viewing, updated by user actions

View:

Presentation of data in model. Same model may be presented in multiple views E.g. Sales data as table or pie chart Traps user events requesting actions

Controller:

Coordinates model and view Responds to view events Executes queries on model, returns results to view
